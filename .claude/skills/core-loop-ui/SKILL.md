# core-loop-ui — 핵심 사전 UI

목표: **30초~2분 안에 끝나는 부드러운 정리 루프** + "오 편하다"(Aha) 순간. iOS는 정리 티가 약하니 완료·streak로 성취감을 만든다.

## 핵심 사전 (이 흐름을 먼저 완결)
온보딩 → 홈 → 정리(스와이프) → 완료 → (다음 날) 홈.

## 1. 온보딩 (F-07)
가치 설명 → 사진/영상 권한 → 알림 권한 → 알림 시각 설정 → 첫 정리 몇 장 체험까지 **끊김 없이**. 프라이버시 문구("사진은 폰 밖으로 안 나감")를 여기서 노출(F-13).

## 2. 홈 (F-02 진입)
- "오늘 미분류 N장", 큰 **정리 시작** 버튼, 연속 정리일(streak).
- N은 platform-integrator의 `loadUnsorted` 결과 개수. 0장이면 빈 상태(성취 메시지).

## 3. 정리 = 스와이프 배정 (F-03, 핵심)
- 한 번에 한 자산(큰 썸네일). **스와이프 제스처로 앨범 배정 / 건너뛰기**, 다음 항목 자동 이동.
- 앨범 선택 모달(F-04): 기존 앨범 리스트 + 인라인 새 앨범 생성.
- 건너뛰기(F-11): 다음 회차에 다시 등장(기록하지 않음).
- 배정 동작은 `assign()` 인터페이스 호출로만. 성공/실패 피드백 즉시.
- **광고를 이 흐름 중간·사진 사이에 절대 넣지 않는다.**

## 4. 영상 지원 (F-08)
- 영상 썸네일 + 재생 아이콘, 탭 시 가벼운 인라인 미리보기. 풀 플레이어·편집은 범위 밖.

## 5. 완료 (F-09 자리, 성취 보상)
- "정리 완료! N장" + streak 증가 시각화 + 처치(간단 애니메이션). iOS 성취감 보완의 핵심.
- 광고는 **여기 화면 뒤에서만**, 설치 7일 후, 세션당 1회(monetization-and-analytics 스킬 규칙 준수).

## 상태관리·구조
- Riverpod로 화면별 상태(로딩/빈/정상/에러) 관리. 화면은 core 서비스 인터페이스에만 의존, 플랫폼 API 직접 호출 금지.
- 썸네일은 캐싱·페이지네이션으로 스크롤 부드럽게(PRD 성능 요구).

## 검증 포인트 (QA에 넘길 것)
- 배정/건너뛰기 후 다음 항목으로 정확히 이동하는가, 마지막 항목 후 완료로 전이하는가.
- 홈의 N과 실제 미분류 개수가 일치하는가.
- 광고가 정리 흐름 중간에 뜨지 않는가.
