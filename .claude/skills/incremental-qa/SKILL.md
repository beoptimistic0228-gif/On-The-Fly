# incremental-qa — 통합 정합성 검증

"파일 있음"이 아니라 **"경계면이 서로 맞는가"**를 본다. 가장 흔한 버그는 shape 불일치다.

## 방법: 경계면 교차 비교
한쪽 코드만 보지 말고, **연결된 양쪽을 동시에 읽어** 대조한다.

| 경계면 | 대조 항목 |
|--------|----------|
| 사진 서비스 반환 ↔ UI 기대 | `loadUnsorted`가 주는 필드·타입·null ↔ 화면이 읽는 필드 |
| DB 스키마 ↔ 리포지토리 쿼리 | 컬럼명·타입 ↔ INSERT/SELECT |
| `assign()` 계약 ↔ UI 호출 | 인자 순서·비동기·에러 타입 ↔ 호출부 |
| 분석 이벤트 정의 ↔ 발생 코드 | 이벤트명·속성 스펠링 ↔ 실제 로깅 |

## 핵심 불변식 (반드시 검증)
1. **중복방지**: 배정한 자산이 `loadUnsorted` 재호출에서 빠지는가.
2. **미분류 정확성(iOS)**: 원본이 타임라인에 남아도 처리 ID 기준으로 재등장 안 하는가.
3. **판별 기준**: 미분류 판별이 OS 앨범 소속이 아니라 처리 ID 집합으로 되어 있는가(코드 확인).
4. **광고 위치**: 정리 흐름 중간·사진 사이에 광고 삽입이 없는가. 7일·세션당 1회 규칙.
5. **권한 엣지**: 거부/제한(iOS limited) 상태에서 크래시 없이 안내로 이어지는가.
6. **프라이버시**: 원본 바이트를 앱 외부로 내보내는 경로가 없는가.

## 실행 원칙
- **점진적**: 전체 완성 후 1회가 아니라 각 모듈 완성 직후.
- 가능하면 `flutter analyze`·단위 테스트로 재현 가능한 근거 확보. 못 돌리면 정적 대조로 폴백하고 한계를 명시.
- 발견은 심각도순으로 `_workspace/03_qa_report.md`에: **경계면 / 재현 / 수정 제안**. 상충 데이터는 삭제 말고 출처 병기.
- 이슈는 해당 에이전트에게 직접 통신해 수정 요청, 수정 후 재검증.
