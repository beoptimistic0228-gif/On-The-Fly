# monetization-and-analytics — 수익화·분석

모두 **서버 없이** SDK/스토어로 처리(PRD 7절). MVP 필수는 분석(F-12), 광고·IAP는 P1(F-09/F-10).

## 1. 광고 (F-09) — AdMob (`google_mobile_ads`)
**노출 규칙(엄격히):**
- 설치 후 **7일 경과** 전에는 광고 없음(첫 주 무료).
- **"정리 완료" 화면 뒤에서, 세션당 1회만.**
- 정리 흐름 중·사진 사이 삽입 **절대 금지**(정리 습관 방해 = 최대 리스크).
- 노출 트리거 기준(설치일 vs 첫 정리일)은 PRD 오픈이슈 → 확정 전 사용자 확인.

## 2. 광고 제거 IAP (F-10) — `in_app_purchase`
- 일회성 구매로 광고 영구 제거. **구매 복원** 지원(재설치·기기변경).
- 구매 상태는 로컬 저장 + 스토어 영수증으로 검증(서버 없음).

## 3. 분석 (F-12) — MVP 필수
North Star(주간 정리 완료자)·D7 리텐션·세션당 미분류 감소를 산출할 **최소 이벤트 세트**:

| 이벤트 | 시점 | 속성 |
|--------|------|------|
| `app_open` | 앱 실행 | (기본) |
| `onboarding_complete` | 온보딩 끝 | 알림시각 |
| `sort_session_start` | 정리 시작 | 미분류수 |
| `asset_assigned` | 배정 성공 | albumId |
| `asset_skipped` | 건너뛰기 | — |
| `sort_session_complete` | 완료 화면 | 처리수, 남은 미분류수 |
| `notification_opened` | 알림 탭 진입 | — |

- 이벤트 스키마를 먼저 확정하고 코드 전반에 일관되게 심는다. 이름·속성 오타는 지표를 망가뜨린다.
- SDK는 Firebase Analytics 또는 Amplitude(둘 다 무료 티어). 선택 근거를 노트에 남긴다.

## 프라이버시
분석에 사진 내용·개인정보를 보내지 않는다. 카운트·이벤트만. PRD "최소 수집" 준수.

## 검증 포인트 (QA에 넘길 것)
- 설치 7일 전/후 광고 노출 여부가 규칙대로인가.
- 광고가 완료 화면 뒤에서만 뜨는가.
- 핵심 이벤트가 실제 발생 시점에 정확한 속성으로 찍히는가.
