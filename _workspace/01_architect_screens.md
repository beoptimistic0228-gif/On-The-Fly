# 01 · 화면 스펙 (Screens)

> SSOT: `그때그때_PRD_v1.0.md` 5.5절. 화면 6개.
> 이 문서는 **feature-builder가 화면을 짤 때 그대로 보는 계약서**다. 각 화면마다 **구성 요소 / 화면 상태 / 진입·이탈 전이 / 사용자 액션**을 표로 정리한다.
> 용어 풀이(초심자용): **화면 상태** = 같은 화면이 데이터 상황에 따라 다르게 보이는 모습(로딩 중 / 데이터 없음 / 정상 / 에러). Flutter에선 보통 `AsyncValue`(Riverpod) 같은 걸로 이 4가지를 스위치한다.

---

## 결론 먼저 — 핵심 사전(single journey) 한 장 요약

```
[온보딩] --권한·알림시간 설정--> [홈] --"정리 시작"--> [정리(스와이프)]
                                                          |  ↕ (앨범 선택 모달)
                                                          | 큐 소진
                                                          v
                                                       [완료] --닫기--> [홈]
[홈] --톱니--> [설정]
```

- **MVP가 완성해야 할 단 하나의 여정**: 권한 허용 → 첫 알림 → 미분류 큐 → 스와이프로 몇 장 배정(= Aha moment) → "정리 완료!" → 다음 날 재방문.
- 아래 순서로 읽어라: **1) 온보딩 → 2) 홈 → 3) 정리 → 5) 완료** 가 핵심 루프다. **4) 앨범 선택**은 정리 화면의 부속 모달, **6) 설정**은 부가.

---

## 1. 온보딩 (Onboarding) — `features/onboarding`

첫 실행 때만 보이는 3~4스텝 시퀀스. 목적: 가치 전달 + 권한 획득 + 알림 시간 설정 + 첫 정리로 자연스럽게 넘김(F-07).

### 구성 요소
| 스텝 | 요소 | 비고 |
|------|------|------|
| 1. 가치 소개 | 로고, 한 줄 카피("하루 한 번, 스와이프로 정리"), "사진은 폰 밖으로 안 나갑니다" 프라이버시 문구(F-13), [시작] 버튼 | 프라이버시 = 마케팅 메시지, 반드시 노출 |
| 2. 사진/영상 권한 | 설명 + [사진 접근 허용] 버튼 → OS 권한 다이얼로그 | iOS는 **전체 접근** 필요 안내 |
| 3. 알림 권한 + 시간 | 설명 + [알림 허용] 버튼 → OS 다이얼로그, 이어서 시간 피커(기본 예: 21:00) | F-02 |
| 4. 첫 정리 유도 | "오늘 미분류 N장이 있어요" + [지금 첫 정리 시작] | 바로 정리 화면으로 |

### 화면 상태
| 상태 | 조건 | 표시 |
|------|------|------|
| 정상 | 첫 실행 | 스텝 시퀀스 |
| 권한 거부 | 사진/알림 거부 | 부드러운 재요청 카드 + "설정 앱에서 켜기" 딥링크. 사진 거부 시 핵심 기능 불가 안내 |
| 부분 권한(제한 접근, iOS/Android) | 일부 사진만 허용 | **전체 접근 유도(확정, D2)**: "전체 미분류를 정리하려면 전체 접근이 필요해요" 안내 카드 + 설정 딥링크/재선택 유도. **부분 접근 정리는 MVP 미지원** |
| 에러 | 권한 API 실패 | 재시도 버튼 |

### 진입 · 이탈 전이
- **진입**: 앱 최초 실행 시 `onboardingCompleted == false` 이면 여기로 라우팅.
- **이탈**: 마지막 스텝 완료 → `onboardingCompleted = true` 저장 → **정리(스와이프)** 또는 **홈**.
- 이후 실행부터는 온보딩을 건너뛰고 바로 홈.

### 사용자 액션
- 권한 허용/거부, 알림 시간 선택, [시작]/[첫 정리 시작] 탭, 스텝 스킵(뒤로).

## 2. 홈 (Home) — `features/home`

매일 돌아오는 허브. "오늘 얼마나 밀렸나 + 얼마나 꾸준한가"를 한눈에.

### 구성 요소
| 요소 | 설명 |
|------|------|
| 미분류 카운트 | "오늘 미분류 **N장**" (사진+영상 합계) |
| 정리 시작 버튼 | 큰 CTA. 탭 → 정리 화면 |
| streak(연속 정리일수) | "🔥 N일 연속" — iOS 성취감 리스크 보완용(PRD 10절) |
| 설정 진입 | 우상단 톱니 아이콘 |
| (선택) 마지막 정리 시각 | "어제 12장 정리함" 류 |

### 화면 상태
| 상태 | 조건 | 표시 |
|------|------|------|
| 로딩 | 큐 계산 중(사진 라이브러리 스캔) | 스켈레톤 카운트, 버튼 비활성 |
| 정상 | 미분류 N>0 | 카운트 + 활성 [정리 시작] |
| 빈(Empty) | 미분류 0 | "오늘은 다 정리했어요 🎉" + streak 강조, 버튼 비활성/축하 톤 |
| 에러 | 라이브러리 접근 실패/권한 회수 | 재시도 + 권한 안내 카드 |

### 진입 · 이탈 전이
- **진입**: 온보딩 완료 후 / 알림 탭 후 / 완료 화면 닫은 뒤 / 앱 재실행(온보딩 done).
- **이탈**: [정리 시작] → 정리 화면. 톱니 → 설정.
- **재진입 시 큐 갱신**: 홈에 올 때마다 미분류 큐를 다시 계산(마지막 처리 이후 신규분).

### 사용자 액션
- [정리 시작] 탭, 설정 진입, (pull-to-refresh로 큐 재계산 — 선택).

## 3. 정리 (스와이프) — `features/sort`

MVP의 심장. 미분류 큐를 한 장씩 스와이프로 앨범에 **배정 예약(stage)** 하고, 세션 끝에 **일괄 확정(commit)**(F-03, D1 배치 스테이징 모델). 30초~2분 내 완료 목표.

> **배치 스테이징 모델(D1 확정)**: 스와이프는 즉시 반영이 아니라 "이 사진 → 이 앨범" **예약(stage)**. 세션 끝에 한 번에 **commit**(Android=시스템 동의창 1회 후 배치 이동, iOS=앨범 태깅 즉시)하고, **성공분만** 처리 기록. 데이터 흐름 = datamodel §7.

### 구성 요소
| 요소 | 설명 |
|------|------|
| 카드 스택 | 현재 자산(큰 썸네일). 영상이면 재생 아이콘 + 길이(F-08) |
| 진행 표시 | "3 / 20" 또는 프로그레스 바 |
| 스와이프 제스처 | 좌/우/상 등으로 **배정 예약(stage)**·건너뛰기(방향 매핑은 아래) |
| 앨범 배정 트리거 | 스와이프 시 **앨범 선택 모달** 오픈, 또는 최근 앨범 퀵버튼 |
| **스테이징 상태 표시** | 하단에 "옮길 N장 대기 중" 등 아직 미확정(stage)임을 명시 |
| **정리(commit) 버튼** | 예약분을 실제 반영. Android는 여기서 시스템 동의창 1회. (큐 소진 시 자동 commit 트리거도 가능) |
| 건너뛰기 | [나중에] — 큐에 다시 등장(F-11), stage 안 함 |
| **되돌리기(Undo)** | 방금 **stage 예약** 취소(commit 전이라 시스템 반영 전에 안전하게 취소) |
| 영상 미리보기 | 탭 시 간이 재생(풀 플레이어·편집 없음, F-08) |

> **스와이프 방향 매핑 = 구현 중 조정**(00_decisions 미결, 착수 차단 아님). 잠정안: **오른쪽=앨범 배정 예약(모달)**, **왼쪽=나중에(건너뛰기)**, **위=최근 앨범으로 예약(stage)**. 배정은 대개 "어느 앨범?"을 물어야 하므로, 순수 스와이프 방향만으로 특정 앨범을 정하긴 어렵다 → 모달 병행이 현실적. 어느 경우든 스와이프는 **예약(stage)**, 실제 반영은 세션 끝 commit.

### 화면 상태
| 상태 | 조건 | 표시 |
|------|------|------|
| 로딩 | 다음 페이지 썸네일 로드 | 카드 placeholder(페이지네이션·캐싱, 성능 NFR) |
| 정상 | 큐에 항목 있음 | 카드 스택 + 진행 + "옮길 N장 대기 중"(stage 카운트) |
| commit 중 | 정리 버튼/큐 소진 후 배치 확정 | Android=시스템 동의창 대기, iOS=즉시. 진행 스피너 |
| 빈/완료 | 큐 소진 → commit 수행 | commit 후 자동으로 **완료 화면**으로 전이(commit 결과 전달) |
| 부분 성공 | commit 성공 N/실패 M | 성공분은 처리 기록·완료 반영, **실패 M장은 stage 큐에 유지** + 인라인 안내 "M장 반영 실패, 다시 시도" |
| commit 취소 | 사용자가 동의창 취소 | 전량 미반영, 예약 유지, "정리를 완료하려면 동의가 필요해요" 안내 |
| 권한 회수 | 도중 접근 상실 | 안내 후 홈으로 |

### 진입 · 이탈 전이
- **진입**: 홈 [정리 시작] / 온보딩 4스텝.
- **모달 왕복**: 정리 ↔ **앨범 선택 모달**(배정 대상 고를 때).
- **이탈(정상)**: 큐 소진 또는 [정리] 버튼 → **commit** → **완료** 화면(commit 결과 전달).
- **이탈(중단)**: 뒤로가기 → 홈. 아직 commit 안 한 stage 예약은 유지(다음 진입 시 이어서 commit 가능). 이미 commit·기록된 항목은 확정 유지.

### 사용자 액션
- 스와이프(배정 예약/건너뛰기), 앨범 선택, 새 앨범 생성(모달 경유), 영상 탭 재생, Undo(예약 취소), [정리] commit, 뒤로가기.

### 세션 처리의 내부 순서(중요 — QA 검증 지점, D1 stage→commit)
```
스와이프 → 앨범 선택 → 배정 예약(stage, 메모리/임시 큐에만)  ...반복...
     ↓ (큐 소진 또는 [정리] 버튼)
commit 배치 확정:
  Android → 시스템 동의창 1회 → 배치 이동(moveAssetsToPath)
  iOS     → 앨범 태깅 즉시
     ↓  BatchAssignResult(성공 id / 실패 id / 취소)
  ├ 성공분 → (F-06) ProcessedAsset 기록(Android는 이동 후 최종 id) → 완료 화면 반영
  └ 실패/취소분 → 기록 안 함, stage 큐에 유지, 인라인 안내
```
> **원칙**: "commit 성공"과 "처리 기록"을 한 트랜잭션처럼 묶는다. stage만 된 자산은 아직 미처리이며, 반영 안 됐는데 기록되면 사진이 어디에도 안 담긴 채 큐에서만 사라지는 유실이 생긴다. 상세 흐름은 datamodel §7.

## 4. 앨범 선택 (모달) — `features/album` (Bottom Sheet)

정리 화면 위에 뜨는 모달. 배정 대상 앨범을 고르거나 즉석 생성(F-04).

### 구성 요소
| 요소 | 설명 |
|------|------|
| 기존 앨범 리스트 | 앨범명 + 커버 썸네일 + (선택)개수. 최근 사용 순 정렬 권장 |
| 새 앨범 생성 | [+ 새 앨범] → 이름 입력 → 즉시 목록 상단에 추가 후 선택 |
| 검색/필터 | (앨범 많을 때) 이름 검색 — 선택적 |
| 닫기 | 배정 취소하고 정리로 복귀 |

> **평면 구조(단일 계층)**만. 중첩 폴더·다중 배정은 Out of Scope(PRD 8절) — 넣지 말 것.

### 화면 상태
| 상태 | 조건 | 표시 |
|------|------|------|
| 로딩 | 앨범 목록 로드 | 리스트 스켈레톤 |
| 정상 | 앨범 ≥1 | 리스트 + [+ 새 앨범] |
| 빈 | 앨범 0(첫 사용) | "첫 앨범을 만들어보세요" + 생성 입력 강조 |
| 에러 | 앨범 생성/조회 실패 | 인라인 에러 + 재시도 |

### 진입 · 이탈 전이
- **진입**: 정리 화면에서 배정 트리거(스와이프/버튼).
- **이탈**: 앨범 선택 → 모달 닫고 정리 화면에서 반영 진행. / 닫기 → 배정 취소.

### 사용자 액션
- 앨범 선택, 새 앨범 이름 입력·생성, 검색, 닫기.

## 5. 완료 (Done) — `features/done`

정리 세션의 보상 화면. 성취감 + (조건부) 광고 + 업셀(F-09/F-10).

### 구성 요소
| 요소 | 설명 |
|------|------|
| 축하 | "정리 완료! **N장**" + 애니메이션. **N = commit 성공분 기준**(stage 예약 수 아님) |
| (부분 성공 시) 안내 | commit에서 실패한 M장이 있으면 "M장은 다시 시도해요" 보조 문구(성공 N만 축하) |
| streak 시각화 | "🔥 N일 연속" 갱신 — iOS 성취감 보완(PRD 리스크) |
| 광고 슬롯 | **첫 정리일 + 7일**부터(확정, D3) **세션당 1회**만, 축하 **뒤**에 노출(F-09). 정리 중·사진 사이 삽입 금지 |
| 광고 제거 업셀 | "광고 없이 쓰기" → IAP(F-10) |
| 닫기/계속 | [홈으로] |

### 화면 상태
| 상태 | 조건 | 표시 |
|------|------|------|
| 정상(광고 없음) | 첫 정리일 +7일 이내 or 광고제거 구매자 or 세션 내 이미 1회 노출 | 축하 + 업셀만 |
| 정상(광고 있음) | 첫 정리일 +7일 경과 & 미구매 & 세션 첫 완료 | 축하 → 광고 1회 → 업셀 |
| 로딩 | 광고 로드 대기 | 광고 자리 placeholder(짧게), 실패 시 스킵 |
| 에러(광고 실패) | 네트워크/노필 | 광고 조용히 스킵, 축하·업셀은 정상 |

> **광고 노출 트리거 = 첫 정리일 + 7일 확정(D3).** 설계는 트리거 계산을 한 곳(core의 정책 함수 `shouldShowAd()`)에 캡슐화한다.

### 진입 · 이탈 전이
- **진입**: 정리 화면 큐 소진 → commit 완료(commit 결과가 완료 화면으로 전달).
- **이탈**: [홈으로] → 홈(갱신된 카운트 0 + streak).

### 사용자 액션
- 광고 시청(수동 개입 없음), [광고 제거 구매], [홈으로].

## 6. 설정 (Settings) — `features/settings`

부가 화면. 알림·결제·프라이버시.

### 구성 요소
| 요소 | 설명 |
|------|------|
| 알림 시간 | 시간 피커 재설정(F-02), 알림 on/off |
| 광고 제거 | [구매] / [구매 복원](F-10) |
