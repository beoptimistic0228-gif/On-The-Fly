# 그때그때 — 공통 디자인 원칙

> 출처: 화면 와이어프레임 기획(v0.2, 2026-07-02)에서 흡수. 와이어프레임 자체는 구현(`lib/features/`)이 SSOT가 되어 폐기하고, **화면이 바뀌어도 유지해야 할 "왜"**만 남긴다.
> 새 화면·화면 개편 시 이 4원칙을 체크리스트로 쓴다.

## 4원칙

1. **사진이 주인공** — 정리 화면에서 사진·영상 카드가 화면의 60%를 차지한다. "보면 바로 안다"는 사용자 특성을 살려, 맥락(날짜·영상 길이)만 얹는다.
2. **배정은 탭 한 번** — 반복 노동을 줄이는 핵심. 앨범을 하단 칩으로 깔아 한 번에 던진다. 판단(사진 보기)→행동(칩 탭)을 최단 경로로 묶는다.
3. **streak로 습관 자극** — 연속 정리일수를 상단·완료 화면에 노출해 매일 오게 만든다. (iOS "앨범=태그라 성취감 약함" 리스크의 보완 장치이기도 하다.)
4. **보상 먼저, 광고 나중** — 광고는 "완료(축하)" 뒤에만. 정리 흐름 중간엔 절대 넣지 않는다. 좋은 행동(정리)의 순간에 벌(광고)을 주지 않기 위함.

## 화면별 존재 이유 (한 줄씩)

| 화면 | 이 화면의 단 하나의 목표 |
|------|--------------------------|
| 온보딩 | 권한을 **납득시키고**(permission priming — 왜 필요한지 먼저, 요청은 뒤에) 알림 시간을 정한다. 이탈 최대 구간이라 마찰 최소화. |
| 홈 | "정리 시작" 버튼을 누르게 한다. 미분류 숫자로 동기 부여, CTA 최대 크기. |
| 정리(스와이프) | 앱의 심장. 항목당 배정을 탭 1회로. |
| 완료 | 축하·통계·streak 보상 → **그 아래** 광고(설치 7일 후) → 광고 제거 업셀. |
| 앨범 선택(모달) | 하단 칩은 자주 쓰는 앨범 전용, 앨범이 많아지면 이 모달이 전체+검색+생성 담당. |
| 설정 | 로그인·프로필·동기화 항목이 **아예 없음** = 단순함이자 프라이버시 강점. "사진은 폰 밖으로 안 나감"을 여기서도 반복 노출. |
